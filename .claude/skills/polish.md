# polish

Review the dashboard for visual rough edges and propose fixes.

## Steps
1. Read `index.html` and examine the current layout, colors, labels, spacing,
   and responsiveness.
2. Produce a crisp, numbered plan of visual-only improvements. No new features.
   Each item should state the problem and the fix in one sentence.
3. Wait for my approval before making any changes.

## Rules
- Visual changes only: layout, colors, labels, spacing, typography, responsiveness.
- Never add new features, new sections, or new functionality.
- Keep vanilla HTML/CSS/JS — no frameworks or libraries.
- Preserve the existing design language (dark theme, CSS variables, card-based layout).
